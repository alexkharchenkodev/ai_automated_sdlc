# Definition Of Ready

A task is ready when it has:

- clear user or business goal
- explicit primary target
- affected area from the project profile
- acceptance criteria
- non-goals or protected surfaces
- known dependencies
- expected validation evidence
- required human approvals identified

Not ready examples:

- "Make it better."
- "Improve performance" without target metric or surface.
- "Use the new design" without a design source or acceptance rubric.
- "Fix tests" without failing command or failure output.
